Du bist auf dem Linux-Produktionsserver von "therese" (Django-Projekt) und hast Zugriff auf das Dateisystem und die Shell.

**Aktuelles Problem:**
Der User möchte `git pull` ausführen, um den aktuellen Stand (inkl. neuer Gruppen-Logik) zu holen, bekommt aber diesen Fehler:

error: Your local changes to the following files would be overwritten by merge:
        therese/settings/base.py
Please commit your changes or stash them before you merge.
Aborting

**Hintergrund:**
- In der Codebase wird `ALLOWED_HOSTS` bewusst über eine Umgebungsvariable aus einer `.env`-Datei gesteuert (siehe therese/settings/base.py und start_therese_prod.sh).
- Der User hat früher wahrscheinlich manuell `ALLOWED_HOSTS` direkt in `therese/settings/base.py` auf dem Server angepasst (z.B. für 172.26.70.* oder den Server-IP/FQDN).
- Deshalb blockiert der lokale Diff jetzt den Pull.
- Das neueste Code-Update enthält u.a.:
  - Den neuen Management-Befehl `python manage.py ensure_groups`
  - Verbesserungen in `apps/accounts/permissions.py` (Gruppen-Erstellung + Aufräumen alter Gruppen)
  - Automatisches Setup der Gruppen in `start_therese_prod.sh` nach dem migrate
  - Bessere Behandlung von ALLOWED_HOSTS (mit Strip und Kommentar, dass .env empfohlen wird)

**Deine Aufgabe:**
1. Untersuche den aktuellen Zustand auf dem Server.
2. Finde heraus, was genau an `therese/settings/base.py` geändert wurde.
3. Führe einen sicheren Pull durch, ohne dass wichtige Einstellungen verloren gehen.
4. Stelle sicher, dass danach die neuen Gruppen (Employees - View, Assisting Admins, Working Groups - Manage etc.) korrekt angelegt und alte Gruppen entfernt werden.
5. Gib dem User klare, ausführbare Befehle.

**Was du prüfen solltest (in dieser Reihenfolge):**
- `git status`
- `git diff therese/settings/base.py`
- Inhalt von `therese/settings/base.py` (besonders der ALLOWED_HOSTS-Bereich)
- Existiert eine `.env` Datei? Was steht drin?
- Inhalt von `start_therese_prod.sh` (Abschnitt wo .env geladen wird)
- Ob schon Dateien unter `apps/accounts/management/commands/` existieren

**Empfohlener Lösungsweg:**
- Die Änderung sollte idealerweise aus `base.py` raus und in eine `.env`-Datei auf dem Server (die schon vom start-Script geladen wird).
- Verwende `git checkout -- therese/settings/base.py` (oder stash), um den Pull freizumachen.
- Nach dem Pull: `.env` anlegen/bearbeiten mit einem sinnvollen `ALLOWED_HOSTS=...`
- Danach `python manage.py ensure_groups` ausführen (oder den sicheren python -c Fallback).
- Am Ende den Server neu starten (start_therese_prod.sh), damit auch das neue Auto-Setup in der Shell läuft.
- Zeige dem User danach die aktuelle Gruppen-Liste.

Arbeite schrittweise, zeige immer die Befehle, die der User kopieren kann, und erkläre kurz, warum du etwas machst. Sei vorsichtig mit dem Überschreiben von Dateien.
